# Слои приложения

Проект построен по принципу **чистой архитектуры** с чётким разделением ответственности. Все слои изолированы, зависимости направлены внутрь.

| Слой               | Назначение                                                                   |
|--------------------|-------------------------------------------------------------------------------|
| **cmd/**           | Точка входа (`main_web.go`). Запускает DI-контейнер fx.                       |
| **internal/di**    | Модули fx, которые регистрируют конструкторы всех зависимостей (синглтоны).   |
| **internal/web**   | HTTP-слой: хендлеры, DTO, middleware (JWT), маршрутизация, сервер.            |
| **internal/service** | Чистая бизнес-логика (минимакс, валидация, ход). Не знает о БД и HTTP.      |
| **internal/serviceWithRepos** | Декоратор, добавляющий репозиторий (сохранение/загрузку) к чистой логике. |
| **internal/repository** | Интерфейсы и реализация для работы с PostgreSQL (SQL-запросы).               |
| **internal/datasource** | Модели данных для БД, мапперы, пул соединений.                              |
| **internal/domain** | Бизнес-сущности (Game, User, LeaderBoardEntry) и константы.                  |

**Взаимодействие:**
HTTP → web (middleware) → web (handler) → serviceWithRepos → repository → datasource → PostgreSQL

Все компоненты собраны через **fx** – см. [Граф зависимостей (fx)](Граф%20зависимостей%20(fx).md).
