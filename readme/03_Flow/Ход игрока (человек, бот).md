# Ход игрока (человек, бот)

## Ход человека

1. Клиент → `POST /game/{id}/move` с JSON `{"row":1,"col":1}` и заголовком `Authorization`.
2. `GameHandler.HandleMove`:
   - Извлекает `userID` из контекста.
   - Вызывает `GameServiceWithRepo.MakeMovePlayer(gameID, userID, row, col)`.
3. Сервис:
   - Загружает игру.
   - Проверяет статус `playing`.
   - Определяет ожидаемого игрока (по `CurrentPlayer` и `PlayerX_ID`/`PlayerO_ID`).
   - Если `playerID` не совпадает – ошибка.
   - Вызывает `logic.MakeMove` (чистый ход).
   - Если игра завершена, обновляет `Status` и `WinnerID`.
   - Сохраняет игру.
4. Если игра с ботом, после хода человека автоматически вызывается `MakeMoveBot` (если игра не завершена и очередь бота).
5. Возвращает обновлённую игру.

## Ход бота

- Аналогично, но не проверяется `playerID`. Бот выбирает ход через `logic.GetNextMove` и применяет его.

**Важно:** Ход бота вызывается автоматически сервером, клиент отдельно не отправляет.

См. также [Завершение игры и статусы](Завершение%20игры%20и%20статусы.md).
