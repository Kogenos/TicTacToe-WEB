# Лидерборд

## Эндпоинт

- `GET /leaderboard?limit=N` (защищённый, N – число записей, по умолчанию 10, макс. 100).

## Реализация

- `GameHandler.GetLeaderBoard` → `GameServiceWithRepo.GetLeaderBoard` → `GameRepository.GetLeaderBoard`.
- SQL-запрос `leaderboard.sql`:
  - Считает для каждого пользователя победы, поражения, ничьи во всех завершённых играх.
  - Вычисляет `win_ratio = wins / total_games`.
  - Ранжирует по убыванию `win_ratio` и количеству побед.
  - Ограничивает `LIMIT`.

## Ответ

```json
[
  {
    "user_id": "...",
    "login": "alice",
    "wins": 5,
    "total_games": 8,
    "win_ratio": 0.625
  },
  ...
]
```

**Важно:** Пользователи без завершённых игр имеют `wins=0`, `total_games=0`, `win_ratio=0.0`.
