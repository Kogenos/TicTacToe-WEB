# Минимакс
Алгоритм, используемый для выбора оптимального хода в игре с нулевой суммой (крестики-нолики). В проекте реализован в `MinimaxGameService`.
## Принцип работы
- Рекурсивно перебирает все возможные продолжения игры до конечных позиций (победа, поражение, ничья).
- Бот (максимизатор) стремится увеличить оценку, противник (минимизатор) – уменьшить.
- Оценка:
  - победа бота: `WinScore - depth` (10 - глубина, чтобы предпочитать быстрые победы).
  - победа противника: `LoseScore + depth` (-10 + глубина).
  - ничья: 0.
## Функции
- `GetNextMove` – для каждого возможного хода вызывает `minimax` и выбирает лучший.
- `minimax` – рекурсивная оценка.
## Ограничения
- Только для поля 3×3 (глубина ≤ 9). Полный перебор быстрый.
- Для бо́льших полей потребуется альфа-бета отсечение или эвристики.
