# Чёрный список refresh-токенов

## Зачем

Предотвращает повторное использование одного refresh-токена после его обновления. Повышает безопасность.

## Как работает

1. При успешном обновлении (`RefreshAccessToken` или `RefreshRefreshToken`) вычисляется SHA‑256 хеш старого refresh-токена.
2. Хеш и `expires_at` (из `exp`) сохраняются в таблицу `refresh_token_blacklist`.
3. При любой валидации refresh-токена сначала проверяется его наличие в чёрном списке через `IsBlacklisted`.
4. Если токен найден – возвращается ошибка `"refresh token already used"`.

## Преимущества

- Статистическая безопасность: даже если токен скомпрометирован, он становится бесполезным после первого использования.
- Не требует изменения структуры пользователя (добавления поля с последним refresh-токеном).

## Недостатки

- Требуется дополнительная таблица.
- При каждом запросе на обновление нужно выполнять запрос к БД (незначительно).
