# Graceful shutdown

## Как реализовано

- В `di/web.go` функция `RegisterServer` добавляет в `fx.Lifecycle` два хука:
  - `OnStart` – запускает сервер в горутине.
  - `OnStop` – вызывает `srv.Stop(ctx)` с контекстом, имеющим таймаут **5 секунд**.
- `srv.Stop(ctx)` внутри вызывает `http.Server.Shutdown(ctx)`.

## Что происходит при завершении

1. Сервер перестаёт принимать новые запросы.
2. Ожидает завершения уже обрабатываемых запросов до истечения таймаута.
3. Если таймаут истёк, соединения закрываются принудительно.
4. Приложение завершается.

## Зачем

- Не обрывать клиентов, которые отправили запрос.
- Дать возможность завершить транзакции БД.
