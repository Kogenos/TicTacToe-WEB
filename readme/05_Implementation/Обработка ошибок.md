# Обработка ошибок

## В HTTP-слое

- Используется единая функция `sendError(w, message, statusCode)`.
- Ответ всегда JSON: `{"error": message}`.
- Типичные коды:
  - `400` – неверный запрос (невалидный JSON, некорректные параметры).
  - `401` – неавторизован (отсутствует или неверный токен).
  - `404` – игра/пользователь не найдены.
  - `409` – конфликт (логин уже существует).
  - `500` – внутренняя ошибка сервера.

## В сервисах и репозиториях

- Ошибки оборачиваются через `fmt.Errorf("context: %w", err)` для сохранения цепочки.
- В репозиториях отличают `sql.ErrNoRows` (нет записи) от других ошибок.

## Graceful shutdown

- При получении сигнала `SIGINT`/`SIGTERM` сервер вызывает `Shutdown`, чтобы завершить текущие запросы.
- См. [Graceful shutdown](Graceful shutdown.md).

## Валидация входных данных

- Длина логина/пароля проверяется в `UserService`.
- Координаты хода проверяются в `MakeMove`.
- `limit` проверяется в хендлере лидерборда.
